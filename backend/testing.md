# Backend Testing Notes

This document summarizes the API areas that were tested during development. Personal emails, access tokens, refresh tokens, provider file identifiers, and account specific examples have been removed for showcase use.

## User Routes

POST `/api/v1/user/register`

Registers a user and returns a redirect URL for the external account connection flow.

POST `/api/v1/user/login`

Accepts form encoded credentials and returns an application access token.

GET `/api/v1/user/me`

Returns the current user profile when a valid authorization header is provided.

GET `/api/v1/user/token`

Returns connection status only. Raw external service tokens are not returned.

## Course Routes

POST `/api/v1/courses/`

Creates a course for the current user.

GET `/api/v1/courses/`

Returns all courses for the current user.

GET `/api/v1/courses/{course_id}`

Returns one course when it belongs to the current user.

PUT `/api/v1/courses/{course_id}`

Updates a course when it belongs to the current user.

DELETE `/api/v1/courses/{course_id}`

Deletes a course when it belongs to the current user.

GET `/api/v1/courses/{course_id}/tutor-sessions`

Returns tutoring sessions associated with a course owned by the current user.

## File Routes

POST `/api/v1/files/`

Creates a file reference for a course owned by the current user.

GET `/api/v1/files/`

Returns all file references for the current user.

GET `/api/v1/files/{file_id}`

Returns one file reference when it belongs to the current user.

PUT `/api/v1/files/{file_id}`

Updates the file name when the file belongs to the current user.

DELETE `/api/v1/files/{file_id}`

Deletes a file reference when it belongs to the current user.

GET `/api/v1/files/course/{course_id}`

Returns file references for a course owned by the current user.

## Tutor Session Routes

POST `/api/v1/tutor-session/chat`

Creates a tutoring session for a course owned by the current user.

GET `/api/v1/tutor-session/{tutor_session_id}`

Returns one tutoring session when it belongs to the current user.

PUT `/api/v1/tutor-session/{tutor_session_id}`

Updates a tutoring session title when it belongs to the current user.

DELETE `/api/v1/tutor-session/{tutor_session_id}`

Deletes a tutoring session when it belongs to the current user.

GET `/api/v1/tutor-session/{tutor_session_id}/messages`

Returns messages for a tutoring session owned by the current user.

## Chat Message Routes

POST `/api/v1/chat-messages/`

Creates a user message and returns the generated assistant response.

GET `/api/v1/chat-messages/`

Returns chat messages owned by the current user.

GET `/api/v1/chat-messages/{message_id}`

Returns one chat message when it belongs to the current user.

PATCH `/api/v1/chat-messages/{message_id}`

Updates one chat message when it belongs to the current user.

DELETE `/api/v1/chat-messages/{message_id}`

Deletes one chat message when it belongs to the current user.

## Drive Routes

GET `/api/v1/drive/search/`

Searches connected study files for the current user.

GET `/api/v1/drive/search/all`

Returns connected study files for the current user.

POST `/api/v1/drive/read`

Reads a connected file for the current user.

## Video Routes

POST `/api/v1/video/generate`

Generates an educational video from a connected file.

GET `/api/v1/video/my`

Lists videos generated by the current user.

GET `/api/v1/video/{filename}`

Serves a generated video only when the file belongs to the current user.

GET `/api/v1/video/templates/list`

Lists available video templates.

DELETE `/api/v1/video/cleanup`

Runs cleanup for old generated video files.
